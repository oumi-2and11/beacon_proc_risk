## 一、环境说明与连接配置

### 1.1 推荐通过 Docker Compose 启动 MySQL

本项目建议仅将 MySQL 容器化，Flask 应用在宿主机（Win11 开发机）本地运行，两者通过 `127.0.0.1:3306` 互通。在仓库根目录新建或参考 `docker-compose.yml`：

```yaml
services:
  mysql:
    image: mysql:8.4
    container_name: proc_risk_mysql
    restart: unless-stopped
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}   # 不要在此处硬写密码，通过 .env 注入
      MYSQL_DATABASE: beacon_proc_risk
      MYSQL_USER: oumi
      MYSQL_PASSWORD: ${MYSQL_APP_PASSWORD}          # 同上，通过 .env 注入
      TZ: Asia/Shanghai
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
    command: >
      --default-authentication-plugin=mysql_native_password
      --character-set-server=utf8mb4
      --collation-server=utf8mb4_0900_ai_ci

volumes:
  mysql_data:
```

在同目录创建 `.env` 文件（**不要提交到 Git**，已加入 `.gitignore`）：

```dotenv
MYSQL_ROOT_PASSWORD=<你的root密码占位符>
MYSQL_APP_PASSWORD=<你的应用账号密码占位符>
```

启动命令（在仓库根目录执行）：

```bash
docker compose up -d
```

之后可通过 `http://127.0.0.1:8081` 访问可选的 phpMyAdmin 管理界面（需在 `docker-compose.yml` 中额外添加 `phpmyadmin` 服务）。

---

### 1.2 Flask 应用通过 SQLAlchemy 连接

**SQLAlchemy** 是 Python 最主流的 ORM/数据库工具库，可将 MySQL 表映射为 Python 类，无需手写 SQL 字符串，同时支持事务与连接池管理。

在 `config.py` 中添加数据库连接串：

```python
import os

class Config:
    # 从环境变量读取，不在代码中硬写密码
    DB_HOST = os.environ.get("DB_HOST", "127.0.0.1")
    DB_PORT = os.environ.get("DB_PORT", "3306")
    DB_USER = os.environ.get("DB_USER", "oumi")
    DB_PASSWORD = os.environ.get("DB_PASSWORD", "<占位符，通过环境变量注入>")
    DB_NAME = os.environ.get("DB_NAME", "beacon_proc_risk")

    SQLALCHEMY_DATABASE_URI = (
        f"mysql+pymysql://{DB_USER}:{DB_PASSWORD}"
        f"@{DB_HOST}:{DB_PORT}/{DB_NAME}?charset=utf8mb4"
    )
    SQLALCHEMY_TRACK_MODIFICATIONS = False
```

需安装依赖：

```bash
pip install SQLAlchemy PyMySQL flask-sqlalchemy
```

## 二、整体 ER 关系概览

```
scan_runs (1) ─────── (N) scan_targets
                              │
              ┌───────────────┼───────────────┐
              │               │               │
     process_catalog    scan_scores     rule_hits
     (N:1 via process_id)  (1:1)        (1:N)
              │
    ┌─────────┼──────────┐
    │         │          │
net_beacon_stats  net_remote_summary  net_connections
   (1:1)          (1:N)               (1:N)

rules ← rule_hits.rule_id (字符串关联，解耦)
allowlist (独立白名单，不参与 FK)
```

- **`scan_runs`**：每次扫描任务一行，是整个数据链的根节点。
- **`scan_targets`**：每条记录代表"本次扫描关注了哪个进程实例"，桥接 `scan_runs` 与 `process_catalog`。
- **`process_catalog`**：进程静态信息的"去重档案"，避免相同进程在多次扫描中重复冗余字段。
- **`scan_scores` / `rule_hits`**：该进程在本次扫描中的评分与证据，1:1 / 1:N 挂在 `scan_targets` 下。
- **`net_*` 三张表**：网络证据，同样挂在 `scan_targets` 下，支撑 Beaconing 检测与进程详情页展示。
- **`rules`**：规则字典，独立维护，`rule_hits` 通过 `rule_id` 字符串松耦合关联。
- **`allowlist`**：白名单配置，独立表，与其余表无外键约束。

### 1. `scan_runs` — 扫描任务表

**用途**：记录每次扫描任务的元信息（何时扫、扫什么模式、扫了多少进程、发现了多少高危），是 `/scans` 列表页和 `/dashboard` 统计的数据来源。

#### 字段说明

| 字段名         | 类型                                          | 可空 | 默认值                    | 含义                                     |
|----------------|-----------------------------------------------|------|---------------------------|------------------------------------------|
| `id`           | `BIGINT UNSIGNED`                             | 否   | `AUTO_INCREMENT`          | **PK**，自增主键                         |
| `scan_uuid`    | `CHAR(36)`                                    | 否   | —                         | **UK**，对外展示/URL 使用的 UUID         |
| `mode`         | `ENUM('single','multi','full','manual')`      | 否   | `'single'`                | 扫描模式：单进程/多进程/全量/手动        |
| `status`       | `ENUM('queued','running','finished','failed')` | 否   | `'finished'`              | 任务状态                                 |
| `requested_by` | `VARCHAR(64)`                                 | 是   | `NULL`                    | 发起者（预留用户字段）                   |
| `created_at`   | `DATETIME(3)`                                 | 否   | `CURRENT_TIMESTAMP(3)`    | 任务创建时间（毫秒精度）                 |
| `started_at`   | `DATETIME(3)`                                 | 是   | `NULL`                    | 实际开始扫描时间                         |
| `finished_at`  | `DATETIME(3)`                                 | 是   | `NULL`                    | 扫描完成时间                             |
| `duration_ms`  | `INT UNSIGNED`                                | 是   | `NULL`                    | 扫描耗时（毫秒）                         |
| `host_name`    | `VARCHAR(128)`                                | 是   | `NULL`                    | 被扫主机名（支持多主机扩展）             |
| `os_info`      | `VARCHAR(128)`                                | 是   | `NULL`                    | 操作系统版本信息                         |
| `app_version`  | `VARCHAR(32)`                                 | 是   | `NULL`                    | 应用版本号                               |
| `target_count` | `INT UNSIGNED`                                | 否   | `0`                       | 本次扫描进程数                           |
| `high_count`   | `INT UNSIGNED`                                | 否   | `0`                       | 高风险进程数                             |
| `mid_count`    | `INT UNSIGNED`                                | 否   | `0`                       | 中风险进程数                             |
| `low_count`    | `INT UNSIGNED`                                | 否   | `0`                       | 低风险进程数                             |
| `note`         | `VARCHAR(255)`                                | 是   | `NULL`                    | 备注说明                                 |

#### 键与索引

| 类型    | 字段          | 说明                         |
|---------|---------------|------------------------------|
| PK      | `id`          | 自增主键                     |
| UK      | `scan_uuid`   | 对外唯一标识，URL 路径使用   |
| Index   | `created_at`  | 按时间倒序查询列表           |
| Index   | `mode`        | 按扫描模式过滤               |
| Index   | `status`      | 按任务状态过滤               |

| 1  | a1b2c3d4-0001-0001-0001-000000000001 | full   | finished | 187         | 3          | 12        | 172       | 2026-04-10 14:32:11 |
| 2  | a1b2c3d4-0002-0002-0002-000000000002 | single | finished | 1           | 1          | 0         | 0         | 2026-04-11 09:05:43 |

### 2. `process_catalog` — 进程实例档案表

**用途**：存储进程的静态快照信息（路径、命令行、用户、父进程等），避免在多次扫描记录中重复冗余相同字段。供 `/processes/<pid>` 详情页展示基本信息，以及历史风险趋势查询。

#### 字段说明

| 字段名          | 类型                                                | 可空 | 默认值                 | 含义                                                   |
|-----------------|-----------------------------------------------------|------|------------------------|--------------------------------------------------------|
| `id`            | `BIGINT UNSIGNED`                                   | 否   | `AUTO_INCREMENT`       | **PK**，自增主键                                       |
| `pid`           | `INT UNSIGNED`                                      | 否   | —                      | 进程 ID（Windows PID 可复用，不唯一）                  |
| `start_time`    | `DATETIME(3)`                                       | 是   | `NULL`                 | 进程启动时间（psutil `create_time`）                   |
| `process_key`   | `VARCHAR(128)`                                      | 否   | —                      | **UK**，自定义唯一标识，格式如 `{pid}:{start_epoch}`   |
| `name`          | `VARCHAR(260)`                                      | 是   | `NULL`                 | 进程名（如 `python.exe`）                              |
| `exe_path`      | `TEXT`                                              | 是   | `NULL`                 | 可执行文件完整路径                                     |
| `cmdline`       | `TEXT`                                              | 是   | `NULL`                 | 命令行参数                                             |
| `username`      | `VARCHAR(128)`                                      | 是   | `NULL`                 | 运行账号（如 `DESKTOP-XX\you`）                        |
| `ppid`          | `INT UNSIGNED`                                      | 是   | `NULL`                 | 父进程 PID                                             |
| `parent_name`   | `VARCHAR(260)`                                      | 是   | `NULL`                 | 父进程名                                               |
| `is_system`     | `TINYINT(1)`                                        | 否   | `0`                    | 是否为系统进程（用于规则判断）                         |
| `signed_status` | `ENUM('unknown','signed','unsigned','invalid')`     | 否   | `'unknown'`            | 数字签名状态                                           |
| `first_seen_at` | `DATETIME(3)`                                       | 否   | `CURRENT_TIMESTAMP(3)` | 首次被扫描发现的时间                                   |
| `last_seen_at`  | `DATETIME(3)`                                       | 否   | `CURRENT_TIMESTAMP(3)` | 最近一次被扫描发现的时间                               |

#### 键与索引

| 类型  | 字段                    | 说明                                 |
|-------|-------------------------|--------------------------------------|
| PK    | `id`                    | 自增主键                             |
| UK    | `process_key`           | 进程实例唯一标识，防 PID 复用混淆    |
| Index | `pid`                   | 按 PID 快速查找                      |
| Index | `name(64)`              | 按进程名前缀查找                     |
| Index | `last_seen_at`          | 按最近发现时间排序                   |

#### 示例数据

| id | pid  | process_key       | name          | exe_path                              | username          | ppid | signed_status |
|----|------|-------------------|---------------|---------------------------------------|-------------------|------|---------------|
| 1  | 4892 | 4892:1712728800   | python.exe    | C:\Python311\python.exe               | DESKTOP-A1\oumi   | 2108 | signed        |
| 2  | 7731 | 7731:1712732400   | beacon_c2.exe | C:\Users\oumi\AppData\Local\Temp\b.exe | DESKTOP-A1\oumi  | 4892 | unsigned      |
| 3  | 1024 | 1024:1712640000   | svchost.exe   | C:\Windows\System32\svchost.exe       | SYSTEM            | 636  | signed        |

### 3. `scan_targets` — 扫描目标关联表

**用途**：桥接 `scan_runs` 与 `process_catalog`，记录"某次扫描涉及了哪些进程实例"。是后续 `scan_scores`、`rule_hits`、`net_*` 三张表的父节点，对应 `/scans/<scan_id>` 的结果条目列表。

#### 字段说明

| 字段名        | 类型              | 可空 | 默认值                 | 含义                                      |
|---------------|-------------------|------|------------------------|-------------------------------------------|
| `id`          | `BIGINT UNSIGNED` | 否   | `AUTO_INCREMENT`       | **PK**，自增主键                          |
| `scan_run_id` | `BIGINT UNSIGNED` | 否   | —                      | **FK** → `scan_runs.id`（级联删除）       |
| `process_id`  | `BIGINT UNSIGNED` | 否   | —                      | **FK** → `process_catalog.id`             |
| `target_label`| `VARCHAR(64)`     | 是   | `NULL`                 | 可读标签，如 `"pid:7731"` 或 `"beacon_c2.exe"` |
| `collected_at`| `DATETIME(3)`     | 否   | `CURRENT_TIMESTAMP(3)` | 此进程信息采集时间                        |

#### 键与索引

| 类型  | 字段                        | 说明                                       |
|-------|-----------------------------|--------------------------------------------|
| PK    | `id`                        | 自增主键                                   |
| UK    | `(scan_run_id, process_id)` | 同一次扫描中同一进程只记录一次             |
| FK    | `scan_run_id → scan_runs.id`| 级联删除：删除扫描任务时自动清理目标记录   |
| FK    | `process_id → process_catalog.id` | 限制删除：不能删除正在被引用的进程档案 |
| Index | `scan_run_id`               | 按扫描任务快速查询所有目标                 |
| Index | `process_id`                | 按进程查历史扫描记录                       |

#### 示例数据

| id | scan_run_id | process_id | target_label     | collected_at            |
|----|-------------|------------|------------------|-------------------------|
| 1  | 1           | 1          | pid:4892         | 2026-04-10 14:32:15.000 |
| 2  | 1           | 2          | pid:7731         | 2026-04-10 14:32:16.000 |
| 3  | 2           | 2          | beacon_c2.exe    | 2026-04-11 09:05:44.000 |

> `scan_targets.id=2` 和 `id=3` 都指向 `process_catalog.id=2`（同一进程 `beacon_c2.exe`），分别来自第 1 次全量扫描和第 2 次单进程扫描，体现进程被多次关注的历史。

### 4. `scan_scores` — 评分汇总表

**用途**：存储每个扫描目标的综合风险评分（总分、等级、各维度子分、最可疑摘要），是 `/scans/<scan_id>` 结果表格、`/dashboard` TopN 高风险和 `/processes/<pid>` 评分展示的数据来源。

#### 字段说明

| 字段名           | 类型                        | 可空 | 默认值           | 含义                                           |
|------------------|-----------------------------|------|------------------|------------------------------------------------|
| `id`             | `BIGINT UNSIGNED`           | 否   | `AUTO_INCREMENT` | **PK**，自增主键                               |
| `scan_target_id` | `BIGINT UNSIGNED`           | 否   | —                | **UK**，**FK** → `scan_targets.id`（1:1 关系）|
| `total_score`    | `INT UNSIGNED`              | 否   | —                | 综合风险总分（0–100）                          |
| `level`          | `ENUM('LOW','MID','HIGH')`  | 否   | —                | 风险等级（LOW < 40，MID 40–69，HIGH ≥ 70）     |
| `score_process`  | `INT UNSIGNED`              | 否   | `0`              | 进程维度子分（父子关系/路径/签名等）           |
| `score_network`  | `INT UNSIGNED`              | 否   | `0`              | 网络维度子分（Beaconing/可疑远端等）           |
| `score_memory`   | `INT UNSIGNED`              | 否   | `0`              | 内存维度子分（RWX 注入等，可选启用）           |
| `score_other`    | `INT UNSIGNED`              | 否   | `0`              | 其他维度子分                                   |
| `summary_reason` | `VARCHAR(255)`              | 是   | `NULL`           | 最可疑原因一句话摘要，列表页快速展示           |

#### 键与索引

| 类型  | 字段             | 说明                                             |
|-------|------------------|--------------------------------------------------|
| PK    | `id`             | 自增主键                                         |
| UK/FK | `scan_target_id` | 每个扫描目标只有一条评分记录；级联删除           |
| Index | `level`          | 按风险等级筛选（看板 HIGH/MID/LOW 统计）         |
| Index | `total_score`    | 按分数排序（Top 高风险进程）                     |

#### 示例数据

| id | scan_target_id | total_score | level | score_process | score_network | summary_reason                            |
|----|----------------|-------------|-------|---------------|---------------|-------------------------------------------|
| 1  | 1              | 35          | LOW   | 10            | 25            | 有外连但无明显周期性，路径合法             |
| 2  | 2              | 88          | HIGH  | 30            | 58            | 非系统路径+未签名+周期性外连（cv=0.08）   |
| 3  | 3              | 88          | HIGH  | 30            | 58            | 与 scan_target_id=2 同一进程再次高风险    |

### 5. `rules` — 规则字典表

**用途**：将检测规则数据化存储，支撑"规则引擎与评分模型"的可解释性展示。对应 `/export` 中的"规则配置导出"和报告中的"系统可配置"说明。

#### 字段说明

| 字段名           | 类型                                         | 可空 | 默认值 | 含义                                    |
|------------------|----------------------------------------------|------|--------|-----------------------------------------|
| `id`             | `BIGINT UNSIGNED`                            | 否   | `AUTO_INCREMENT` | **PK**，自增主键               |
| `rule_id`        | `VARCHAR(32)`                                | 否   | —      | **UK**，规则编号，如 `"NET-101"`        |
| `title`          | `VARCHAR(128)`                               | 否   | —      | 规则标题（显示用）                      |
| `description`    | `TEXT`                                       | 是   | `NULL` | 规则详细说明                            |
| `dimension`      | `ENUM('process','network','memory','other')` | 否   | —      | 规则所属检测维度                        |
| `default_weight` | `INT`                                        | 否   | `10`   | 命中时默认加分值                        |
| `enabled`        | `TINYINT(1)`                                 | 否   | `1`    | 是否启用（0 = 禁用）                    |

#### 键与索引

| 类型  | 字段        | 说明                       |
|-------|-------------|----------------------------|
| PK    | `id`        | 自增主键                   |
| UK    | `rule_id`   | 规则编号唯一               |
| Index | `dimension` | 按维度筛选规则             |
| Index | `enabled`   | 只查询启用规则             |

#### 示例数据

| id | rule_id  | title                          | dimension | default_weight | enabled |
|----|----------|--------------------------------|-----------|----------------|---------|
| 1  | PROC-001 | 父子进程关系异常               | process   | 20             | 1       |
| 2  | NET-101  | 疑似 Beaconing（周期性外连）   | network   | 35             | 1       |
| 3  | PROC-002 | 可执行文件路径非系统目录且未签名 | process  | 30             | 1       |

### 6. `rule_hits` — 规则命中记录表

**用途**：每命中一条规则存一行，并通过 JSON 字段保存结构化证据（如均值/标准差/父进程名/路径等），构成完整的"证据链"。对应 `/processes/<pid>` 详情页的"规则命中与评分解释"区块，也是导出报告的核心附件数据。

#### 字段说明

| 字段名           | 类型                                         | 可空 | 默认值                 | 含义                                                    |
|------------------|----------------------------------------------|------|------------------------|---------------------------------------------------------|
| `id`             | `BIGINT UNSIGNED`                            | 否   | `AUTO_INCREMENT`       | **PK**，自增主键                                        |
| `scan_target_id` | `BIGINT UNSIGNED`                            | 否   | —                      | **FK** → `scan_targets.id`（级联删除）                  |
| `rule_id`        | `VARCHAR(32)`                                | 否   | —                      | 规则编号（字符串，松耦合关联 `rules` 表）               |
| `title`          | `VARCHAR(128)`                               | 否   | —                      | 规则标题（冗余存储，方便查询时直接展示）                |
| `dimension`      | `ENUM('process','network','memory','other')` | 否   | —                      | 规则维度                                                |
| `score_delta`    | `INT`                                        | 否   | `0`                    | 本条规则对总分的贡献值                                  |
| `evidence_json`  | `JSON`                                       | 是   | `NULL`                 | 证据字段（结构灵活）：如 ppid、均值/标准差、remote_ip 等 |
| `created_at`     | `DATETIME(3)`                                | 否   | `CURRENT_TIMESTAMP(3)` | 规则命中记录时间                                        |

#### 键与索引

| 类型  | 字段             | 说明                               |
|-------|------------------|------------------------------------|
| PK    | `id`             | 自增主键                           |
| FK    | `scan_target_id` | 级联删除                           |
| Index | `scan_target_id` | 查某目标的所有命中规则             |
| Index | `rule_id`        | 按规则统计命中次数                 |
| Index | `dimension`      | 按维度过滤（只看网络/进程规则）    |

#### 示例数据

**scan_target_id = 2（进程 `beacon_c2.exe`，对应 `scan_runs.id=1` 全量扫描）**

| id | scan_target_id | rule_id  | title                             | dimension | score_delta | evidence_json                                                                                  |
|----|----------------|----------|-----------------------------------|-----------|-------------|-----------------------------------------------------------------------------------------------|
| 1  | 2              | NET-101  | 疑似 Beaconing（周期性外连）      | network   | 35          | `{"mean_ms": 30012, "std_ms": 2401, "cv": 0.08, "sample_n": 20, "remote_ip": "203.0.113.55"}` |
| 2  | 2              | PROC-002 | 可执行文件路径非系统目录且未签名  | process   | 30          | `{"exe_path": "C:\\Users\\oumi\\AppData\\Local\\Temp\\b.exe", "signed": false}`                |
| 3  | 2              | PROC-001 | 父子进程关系异常                  | process   | 20          | `{"ppid": 4892, "parent_name": "python.exe", "expected_parent": "explorer.exe"}`              |

> `evidence_json` 使用 MySQL 8 原生 JSON 类型，字段结构可随规则灵活变化，无需改表结构。

### 7. `net_beacon_stats` — Beaconing 统计表

**用途**：存储每个扫描目标的网络连接时间序列统计结果（均值 μ、标准差 σ、变异系数 cv 等），是 Beaconing 检测的核心证据表，对应 `/processes/<pid>` 详情页"网络证据"区块中的 Beaconing 统计展示。

#### 字段说明

| 字段名              | 类型              | 可空 | 默认值           | 含义                                           |
|---------------------|-------------------|------|------------------|------------------------------------------------|
| `id`                | `BIGINT UNSIGNED` | 否   | `AUTO_INCREMENT` | **PK**，自增主键                               |
| `scan_target_id`    | `BIGINT UNSIGNED` | 否   | —                | **UK**，**FK** → `scan_targets.id`（1:1 关系）|
| `sample_window`     | `INT UNSIGNED`    | 否   | `0`              | 采样窗口大小 N（连接次数）                     |
| `interval_mean_ms`  | `DOUBLE`          | 是   | `NULL`           | 连接间隔均值 μ（毫秒）                         |
| `interval_std_ms`   | `DOUBLE`          | 是   | `NULL`           | 连接间隔标准差 σ（毫秒）                       |
| `interval_cv`       | `DOUBLE`          | 是   | `NULL`           | 变异系数 cv = σ/μ（越接近 0 越可疑）           |
| `suspected`         | `TINYINT(1)`      | 否   | `0`              | 是否判定为疑似 Beaconing（1=是）               |
| `jitter_like`       | `TINYINT(1)`      | 否   | `0`              | 是否检测到 Jitter 特征（1=是）                 |
| `notes`             | `VARCHAR(255)`    | 是   | `NULL`           | 补充说明（如"cv 极小，疑似固定间隔心跳"）      |

#### 键与索引

| 类型  | 字段             | 说明                               |
|-------|------------------|------------------------------------|
| PK    | `id`             | 自增主键                           |
| UK/FK | `scan_target_id` | 每个扫描目标仅一条统计；级联删除   |
| Index | `suspected`      | 快速筛选疑似 Beaconing 目标        |

#### 示例数据

| id | scan_target_id | sample_window | interval_mean_ms | interval_std_ms | interval_cv | suspected | notes                     |
|----|----------------|---------------|------------------|-----------------|-------------|-----------|---------------------------|
| 1  | 2              | 20            | 30012.5          | 2401.0          | 0.0800      | 1         | cv 极小，疑似固定间隔心跳 |
| 2  | 3              | 20            | 30015.0          | 2398.5          | 0.0799      | 1         | 二次扫描仍命中 Beaconing  |
| 3  | 1              | 15            | 45230.0          | 18920.0         | 0.4183      | 0         | 波动较大，无明显周期性    |

### 8. `net_remote_summary` — 远端 IP 汇总表

**用途**：对每个扫描目标的出向连接按远端 IP 汇总（连接次数、首末时间、持续时长等），支撑 `/dashboard` 的"Top 可疑远端 IP"统计，以及 `/processes/<pid>` 详情页的"可疑远端汇总"。

#### 字段说明

| 字段名               | 类型                            | 可空 | 默认值     | 含义                                         |
|----------------------|---------------------------------|------|------------|----------------------------------------------|
| `id`                 | `BIGINT UNSIGNED`               | 否   | `AUTO_INCREMENT` | **PK**，自增主键                       |
| `scan_target_id`     | `BIGINT UNSIGNED`               | 否   | —          | **FK** → `scan_targets.id`（级联删除）       |
| `remote_ip`          | `VARCHAR(45)`                   | 否   | —          | 远端 IP（IPv4 或 IPv6）                      |
| `remote_port`        | `INT UNSIGNED`                  | 是   | `NULL`     | 远端端口号（NULL 表示多端口或未知）          |
| `protocol`           | `ENUM('TCP','UDP','UNKNOWN')`   | 否   | `'TCP'`    | 协议类型                                     |
| `conn_count`         | `INT UNSIGNED`                  | 否   | `0`        | 在采样窗口内的连接次数                       |
| `first_seen_at`      | `DATETIME(3)`                   | 是   | `NULL`     | 首次连接时间                                 |
| `last_seen_at`       | `DATETIME(3)`                   | 是   | `NULL`     | 最近连接时间                                 |
| `total_duration_ms`  | `BIGINT UNSIGNED`               | 是   | `NULL`     | 连接总持续时长（毫秒，估算）                 |
| `risk_hint`          | `VARCHAR(128)`                  | 是   | `NULL`     | 风险提示标签，如 `"periodic_traffic"`        |

#### 键与索引

| 类型  | 字段             | 说明                               |
|-------|------------------|------------------------------------|
| PK    | `id`             | 自增主键                           |
| FK    | `scan_target_id` | 级联删除                           |
| Index | `scan_target_id` | 查某目标的所有远端 IP              |
| Index | `remote_ip`      | 跨目标聚合同一远端 IP（看板 TopN） |
| Index | `conn_count`     | 按连接次数排序                     |

#### 示例数据

| id | scan_target_id | remote_ip      | remote_port | protocol | conn_count | risk_hint          |
|----|----------------|----------------|-------------|----------|------------|--------------------|
| 1  | 2              | 203.0.113.55   | 4444        | TCP      | 20         | periodic_traffic   |
| 2  | 2              | 192.0.2.100    | 443         | TCP      | 3          | NULL               |
| 3  | 3              | 203.0.113.55   | 4444        | TCP      | 20         | periodic_traffic   |

> `remote_ip = 203.0.113.55` 同时出现在 `scan_target_id=2`（第一次全量扫描）和 `scan_target_id=3`（第二次单进程扫描），看板 Top 远端 IP 聚合时累计 `conn_count=40`，可直接展示"该 IP 被多次关注"。

### 9. `net_connections` — 连接明细表

**用途**：记录每次扫描时进程的逐条网络连接快照（本地/远端 IP:Port、协议、连接状态），为 `/processes/<pid>` 详情页的"当前连接列表"展示和导出报告的连接明细附录提供数据。

#### 字段说明

| 字段名        | 类型                          | 可空 | 默认值                 | 含义                                       |
|---------------|-------------------------------|------|------------------------|--------------------------------------------|
| `id`          | `BIGINT UNSIGNED`             | 否   | `AUTO_INCREMENT`       | **PK**，自增主键                           |
| `scan_target_id` | `BIGINT UNSIGNED`          | 否   | —                      | **FK** → `scan_targets.id`（级联删除）     |
| `protocol`    | `ENUM('TCP','UDP','UNKNOWN')` | 否   | `'TCP'`                | 协议类型                                   |
| `local_ip`    | `VARCHAR(45)`                 | 是   | `NULL`                 | 本地 IP 地址                               |
| `local_port`  | `INT UNSIGNED`                | 是   | `NULL`                 | 本地端口号                                 |
| `remote_ip`   | `VARCHAR(45)`                 | 是   | `NULL`                 | 远端 IP 地址                               |
| `remote_port` | `INT UNSIGNED`                | 是   | `NULL`                 | 远端端口号                                 |

#### 键与索引

| 类型  | 字段             | 说明                               |

| PK    | `id`             | 自增主键                           |

| Index | `scan_target_id` | 查某目标的所有连接明细             |

#### 示例数据

### 10. `allowlist` — 白名单表

**用途**：存储用户配置的"忽略项"（可信进程路径、进程名、远端 IP、规则 ID），扫描时跳过命中白名单的项目，减少误报。对应 `/processes/<pid>` 详情页"操作区"中的"加入白名单/忽略规则"功能，也用于报告中"预防措施 / 减少误报"章节的说明。

#### 字段说明

| 字段名       | 类型                                                  | 可空 | 默认值                 | 含义                                           |
|--------------|-------------------------------------------------------|------|------------------------|------------------------------------------------|
| `id`         | `BIGINT UNSIGNED`                                     | 否   | `AUTO_INCREMENT`       | **PK**，自增主键                               |
| `type`       | `ENUM('process_path','process_name','remote_ip','rule_id')` | 否 | —              | 白名单类型                                     |
| `value`      | `VARCHAR(512)`                                        | 否   | —                      | 白名单匹配值（路径/进程名/IP/规则编号）        |
| `note`       | `VARCHAR(255)`                                        | 是   | `NULL`                 | 添加原因说明                                   |
| `enabled`    | `TINYINT(1)`                                          | 否   | `1`                    | 是否生效（0 = 临时禁用，不删除）               |
| `created_at` | `DATETIME(3)`                                         | 否   | `CURRENT_TIMESTAMP(3)` | 白名单条目创建时间                             |

#### 键与索引

| 类型  | 字段      | 说明                          |
|-------|-----------|-------------------------------|
| PK    | `id`      | 自增主键                      |
| Index | `type`    | 按类型快速加载对应白名单      |
| Index | `enabled` | 只查询生效的白名单条目        |

#### 示例数据

| id | type         | value                                   | note                         | enabled |
|----|--------------|-----------------------------------------|------------------------------|---------|
| 1  | process_path | C:\Windows\System32\svchost.exe         | 系统服务，不计入风险评分     | 1       |
| 2  | remote_ip    | 8.8.8.8                                 | Google DNS，正常访问         | 1       |
| 3  | rule_id      | NET-101                                 | 测试环境心跳脚本，暂时忽略   | 0       |

> `allowlist.id=3` 的 `enabled=0` 表示已临时禁用（不删除记录，方便日后恢复），体现系统设计的可配置性。

## 附：建表顺序说明

由于存在外键约束，建表时需按以下顺序执行 DDL：

1. `rules`（无外键依赖）
2. `allowlist`（无外键依赖）
3. `scan_runs`（无外键依赖）
4. `process_catalog`（无外键依赖）
5. `scan_targets`（依赖 `scan_runs`、`process_catalog`）
6. `scan_scores`（依赖 `scan_targets`）
7. `rule_hits`（依赖 `scan_targets`）
8. `net_beacon_stats`（依赖 `scan_targets`）
9. `net_remote_summary`（依赖 `scan_targets`）
10. `net_connections`（依赖 `scan_targets`）

完整建表 SQL 可通过 Flask-Migrate（Alembic）或手动在 phpMyAdmin / MySQL CLI 中执行 `schema/` 目录下的初始化脚本创建。
