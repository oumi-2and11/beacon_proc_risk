# 基于 Beacon 行为特征的主机进程异常分析与风险评分系统

---

### 我要做的东西：

我要做的东西是要能检测出本主机中正在运行的进程（包括后台进程），然后我可以选择并进行危险检测扫描。

1. **枚举本机当前所有正在运行的进程（含后台/服务进程）**
2. 在 Web 页面上**展示进程列表**（PID、名称、路径、用户、父进程等）
3. 你从列表里**选择某个/某些进程**
4. 点击“扫描/检测”，系统对所选进程做**危险行为特征检测**（你选题里的：父子拓扑、签名/路径、网络 beaconing、可选内存 RWX）
5. 输出**风险分值 + 等级 + 命中规则 + 证据链**，并能导出/截图写进报告

### 我建议你把功能先拆成两个“必须跑通”的里程碑

#### 里程碑 A（最小可运行：进程枚举 + 选择 + 扫描按钮 + 结果页）

- Flask 页面显示所有进程（后台也包含）
- 支持按 PID 或名称筛选
- 勾选一个进程 → 发起扫描 → 返回结果页/弹窗（至少含：风险等级、总分、命中规则列表）

这一步做完，你就已经能在“执行结果15分”里放大量截图了。

#### 里程碑 B（把检测做成“像安全系统”）

对单个进程做以下检测并打分（从易到难）：

1. **进程静态信息审计**：路径位置、启动时间、父进程、用户、命令行（证据链很好写）
2. **父子进程拓扑异常规则**：比如 explorer→powershell/cmd/python 等（可配置白名单）
3. **网络连接与 beaconing 检测**：采样连接时间序列，计算间隔均值/方差/cv，判断周期性
4. （加分）签名校验、RWX 内存页扫描



* 一些模块的具体实现方式，等做这个模块前再详细设计



### 1. 主机进程行为静态审计模块 (Static Process Auditor)

该模块负责对系统运行中的进程进行“全量体检”，识别出 Beacon 常见的注入载体。

- **父子进程拓扑分析**：检测如 `notepad.exe`、`svchost.exe` 等系统常用进程是否存在异常子进程 。
- **内存属性扫描**：遍历进程内存页，重点识别具有 `PAGE_EXECUTE_READWRITE` (RWX) 权限且不在对应磁盘镜像文件中的异常段，这是 Beacon 内存加载的典型特征。
- **数字签名合规性校验**：自动筛查未签名进程或开发者信息异常的进程，作为风险判定的基础维度。

### 2. 隐蔽通信流量监测模块 (C2 Traffic Monitor)

利用你对 Cobalt Strike Malleable C2 配置的研究，识别网络侧的“蛛丝马迹”。

- **自定义 HTTP 头部审计**：通过正则匹配 User-Agent 异常（如你研究过的特定版本特征）以及 Cookie、Header 中符合 Beacon 指令交换格式的数据 。
- **心跳周期性检测 (Beaconing)**：记录特定进程与远端 IP 通信的时间间隔，通过计算方差识别符合 CS “Sleep + Jitter” 规律的周期性请求。
- **载荷指纹识别**：针对 Stager 或 Beacon 传输过程中的特定数据模式进行扫描。

### 3. 多维加权风险评分引擎 (Risk Scoring Engine)

这是作业要求中“风险评分程序”的核心逻辑所在 ，将零散的证据转化为可量化的结论。

- **风险权重分配**：
	- **内存注入异常**：权重 0.4。
	- **流量特征命中**：权重 0.4。
	- **签名/父子关系异常**：权重 0.2。
- **分级判定逻辑**：系统自动计算总分，将风险判定为“高、中、低”三个等级，并说明判定理由 。

### 4. 风险可视化展示与导出模块 (Visualization & Reporting)

对应作业中的“执行结果”与“风险展示”章节 。

- **实时告警日志**：在控制台或界面实时输出进程 ID (PID)、触发的规则 ID 以及当前的风险分值 。
- **证据链汇总**：自动提取路径、路径、哈希、通信特征等作为“风险证据”。
- **防护建议生成**：根据评分结果，自动从内置库中匹配针对性的缓解措施（如：阻断特定 IP、查杀对应进程、加强目录审计） 。
