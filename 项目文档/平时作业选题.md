# 平时作业选题

## 选题名称

基于 Beacon 行为特征的主机进程异常分析与风险评分系统（ProcRisk）

---

## 选题背景

在主机安全领域，Cobalt Strike 等 C2 框架广泛使用 Beacon 机制控制被植入后门的主机。Beacon 以固定或近似固定的时间间隔向控制服务器发送心跳包（即 Beaconing 通信），这是主机被植入后门最典型的行为特征之一。然而，传统基于特征码或单次快照的检测方法难以识别此类周期性通信行为，且容易被本地代理（如 Clash、Shadowsocks）隐藏真实 C2 地址。

本选题围绕 Beacon 行为特征，设计并实现一套主机进程风险检测系统，通过多轮网络连接采样、CV 统计分析、代理穿透等技术，实现对 Beaconing 通信的有效检测，并结合多维规则引擎对进程进行风险评分。

---

## 选题目标

1. **枚举本机所有运行进程**（含后台/服务进程），采集进程静态信息和网络连接
2. **检测 Beaconing 周期性通信**：通过多轮采样积累真实时间序列，利用变异系数（CV）算法识别 C2 回连规律
3. **代理穿透**：当远端 IP 为本地代理回环地址时，自动追踪代理进程的真实远端连接，还原被隐藏的 C2 地址
4. **多维规则检测与风险评分**：从进程路径、父子关系、签名状态、命名模式、网络行为 5 个维度进行规则匹配，输出量化评分
5. **白名单过滤**：减少系统进程误报，聚焦真正可疑的目标
6. **可视化与报告导出**：Web 界面展示检测过程与结果，支持 JSON 和独立 HTML 报告导出

## 核心技术点

### 1. 多轮网络连接采样

单次 psutil 快照所有连接时间戳相同，无法计算间隔。本系统引入 `BeaconSampler`，以 5 秒间隔采集 12 轮连接快照（共 60 秒），积累真正的时间序列数据，使 CV 算法生效。

### 2. CV 算法检测周期性通信

Beaconing 通信的回连间隔高度规律，变异系数 CV = σ/μ 接近 0。系统从 check-in 时间序列计算间隔的均值和标准差，CV < 0.3 判定为疑似 Beaconing，0.05 < CV < 0.15 判定为含 Jitter 特征。

### 3. 代理穿透

当进程通过本地代理通信时，psutil 报告的远端 IP 为回环地址（如 127.0.0.1:7897）。系统自动查找监听该端口的代理进程，获取其真实远端连接，在结果中标注"经代理 xxx(PID xxx)"并替换为真实 IP。

### 4. 规则引擎 + 类注册机制

采用 `@register_rule` 装饰器 + 全局注册表模式，新增规则只需创建类并装饰，引擎自动发现加载，支持 DB 配置覆盖权重和启用状态。
