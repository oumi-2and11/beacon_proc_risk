# 进程列表智能过滤方案

> **状态：已全部实现**

---

## 背景

扫描一次出现几百条进程，绝大多数是 Windows 系统进程（svchost.exe、csrss.exe 等），用户要在里面找可疑进程非常困难。项目已有 Allowlist 模型、`is_allowed()`、`is_suspicious_path()`、`SUSPICIOUS_PARENT_CHILD` 等代码，但全部未接入管线。

**目标**：进程列表默认只显示"值得调查"的进程，用户可切换查看全部；同时提供白名单管理能力。

## 方案：标记 + 智能默认视图

- 扫描时仍采集全部进程入库（不丢数据），但给每条记录打上风险标签
- 进程列表页默认显示"可疑"视图（排除系统进程+白名单），可切换到"全部"/"高风险特征"
- 提供"白名单"管理页面和进程详情页快速加入白名单功能

## 实施步骤（均已完成 ✅）

### 1. ProcessCatalog 增加两列 ✅

**文件**: `webapp/models.py`

```python
path_suspicious = db.Column(db.Boolean, nullable=False, default=False)
parent_child_suspicious = db.Column(db.Boolean, nullable=False, default=False)
```

`is_system` 和 `signed_status` 已有，不需新增。

### 2. ProcessInfo 增加两字段 ✅

**文件**: `utils/process_collector/models.py`

```python
path_suspicious: bool = False
parent_child_suspicious: bool = False
```

### 3. 采集时接入现有检测器 ✅

**文件**: `utils/process_collector/windows.py`

在 `collect_single_process()` 中：
- 调用 `is_suspicious_path(exe_path)` → `path_suspicious`
- 检查 `SUSPICIOUS_PARENT_CHILD` 映射 → `parent_child_suspicious`
- 修复 `signed_status`：对 `is_system=True` 的进程启发式设为 "signed"（原硬编码 "unknown" 导致过滤无效）
- 将新字段传入 `ProcessInfo()` 构造

### 4. sync_process_catalog 存入新字段 ✅

**文件**: `utils/process_collector/collector.py`

- INSERT 时写入 `path_suspicious`、`parent_child_suspicious`
- UPSERT 时更新这两个字段

### 5. 进程列表增加过滤 Tab ✅

**文件**: `webapp/views/process_view.py`

`list_processes()` 增加 `filter` 参数（默认 `"suspicious"`）：
- `"all"`: 无额外过滤
- `"suspicious"`: 排除 `is_system=True AND signed_status="signed"` 的进程 + 白名单进程
- `"high_risk"`: 仅显示 `path_suspicious=True OR parent_child_suspicious=True OR (signed_status != "signed" AND is_system == False)`

每个 tab 传数量（`total_count`, `suspicious_count`, `high_risk_count`）给模板做 badge。

白名单过滤逻辑：加载 `Allowlist.query.filter_by(enabled=True)`，对每个进程调用 `is_allowed()` 匹配。

搜索和分页链接携带 `filter` 参数。

### 6. 进程详情页传递风险字段 ✅

**文件**: `webapp/views/process_view.py`

`process_detail()` 构建 `proc` dict 时增加：`is_system`, `signed_status`, `path_suspicious`, `parent_child_suspicious`

### 7. 进程列表模板 — Tab + 风险标签 ✅

### 8. 进程详情模板 — 风险指标 + 快速加白名单 ✅

## 与原计划的偏差

| 项目 | 原计划 | 实际实现 | 原因 |
|------|--------|----------|------|
| high_risk 过滤条件 | `signed_status IN ("unsigned","invalid")` | `signed_status != "signed" AND is_system == False` | signed_status 启发式只产出 "signed"/"unknown"，不存在 "unsigned"/"invalid"，原条件永远为空 |
