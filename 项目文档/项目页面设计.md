## 一、整体导航与信息架构

导航栏（`layout.html`）固定 6 个一级入口：

1. 首页
2. 进程列表
3. 扫描中心
4. 风险看板
5. 白名单
6. 导出报告

其中"进程列表"和"扫描中心"是主线；"白名单"管理误报过滤；"风险看板"和"导出报告"用于展示"可见、可测、可感"。

导航栏品牌区包含 `PR` 图标 + "ProcRisk" 文字 + "主机进程风险检测" 副标题，采用 sticky 定位 + 毛玻璃背景效果。

------

## 二、页面清单（共 9 个模板）

### P0. `首页 /`（项目入口与状态概览）

**模板**：`index.html`

**页面结构**：

- **Hero 区域**：大标题渐变色（"主机进程异常分析与风险评分"）+ 副标题说明 + 3 个操作按钮（扫描进程、进程列表、扫描中心）居中排列
- **KPI 卡片**：4 列网格，分别显示已收录进程数、扫描次数、高风险数、中/低风险数（页面加载时通过 `GET /api/kpi` 异步获取）
- **Pipeline 流程条**：5 步水平流程展示功能主线（采集进程 → 多轮采样 → 规则检测 → 风险评分 → 导出报告），带编号圆标和箭头
- **Feature 卡片**：3×2 网格，6 张卡片分别介绍核心功能（Beaconing 采样检测、代理穿透、多维规则引擎、白名单过滤、风险看板、报告导出），每张含图标 + 标题 + 描述，hover 有顶部渐变边框动效

**交互**：点击「扫描进程」按钮调用 `POST /api/scan-processes`，按钮切换加载/成功/失败状态，完成后 Toast 提示并刷新 KPI。

------

### P1. `进程列表 /processes`（枚举 + 智能筛选）

**模板**：`processes.html`

**主要功能**：

- **过滤 Tab**：搜索栏下方 3 个 Tab——全部 / 可疑（默认）/ 高风险特征，各带数量 badge
  - 全部：无额外过滤
  - 可疑：排除 `is_system=True AND signed_status="signed"` 的进程 + 白名单进程
  - 高风险：仅显示 `path_suspicious=True OR parent_child_suspicious=True OR (signed_status != "signed" AND is_system == False)`
- **搜索**：按 PID（精确）或名称（模糊）过滤
- **表格**：PID、名称（含风险标签）、用户、PPID、路径、签名状态、风险标记
  - 名称列风险标签：可疑路径（橙色）、异常父子（橙色）、未签名（红色）、系统（绿色）
- **分页**：每页 50 条，Tab 和搜索参数在分页链接中保持
- **操作**：每行点击进入详情页，详情页可发起扫描

------

### P2. `进程详情 /processes/<pid>`（单进程证据链总览）

**模板**：`process_detail.html`

**主要功能**：

- **基本信息**：PID、名称、路径、命令行、用户、PPID/父进程名、系统进程标记、签名状态、可疑路径标记、异常父子标记
- **操作区**：
  - 「扫描此进程」按钮：异步扫描，点击后 JS 调用 `POST /scans/api/start`，展示采样进度面板（进度条 + 轮次文字），每秒轮询 `GET /scans/api/status/<task_id>`，完成后自动跳转扫描详情
  - 「加入白名单」卡片：两个表单按钮（按名称 / 按路径），POST 到白名单 quick-add 接口
- **最近扫描结果**：从 `ScanRun` 关联查询该 PID 最近一次扫描，展示分数、等级、命中规则

------

### P3. `扫描中心 /scans`（扫描历史记录）

**模板**：`scans.html`

**主要功能**：

- 扫描记录列表（每次 scan_id 一行）：时间、扫描模式、扫描耗时、进程数、风险分布
- 点击进入扫描详情页

------

### P4. `扫描详情 /scans/<scan_id>`（本次扫描完整结果）

**模板**：`scan_detail.html`

**主要内容**：

- **评分总览**：总分 + 等级 badge（绿/橙/红） + 4 个维度子分（process / network / memory / other） + 摘要原因
- **规则命中表**：规则 ID、标题、证据详情
- **Beaconing 检测卡片**：CV、均值、标准差、采样窗口、判定结果（suspected / jitter）、备注
- **远端 IP 汇总表**：IP、端口、协议、连接次数、风险提示（含代理穿透标注如"经代理 verge-mihomo.exe(PID xxx)"）
- **连接明细表**：协议、本地地址:端口、远端地址:端口、状态

------

### P5. `风险看板 /dashboard`（统计 + TopN）

**模板**：`dashboard.html`

**主要内容**：

- KPI 卡片：总扫描数、高风险数、中风险数、低风险数
- 高风险进程表格（TopN）

------

### P6. `白名单 /allowlist`（白名单管理）

**模板**：`allowlist.html`

**主要功能**：

- **添加表单**：类型下拉（process_name / process_path / remote_ip / rule_id）+ 值输入 + 备注输入
- **条目表格**：类型、值、备注、启用状态（开关按钮）、删除按钮
- **默认值**：首次访问自动填充 10 条常见白名单（svchost.exe、csrss.exe 等）
- **操作反馈**：使用 flash messages 提示添加/切换/删除结果

------

### P7. `导出报告 /export`（导出中心）

**模板**：`export.html`

**主要功能**：

- 导出最近一次扫描的 JSON / HTML
- 导出指定 scan_id 的 JSON / HTML
- HTML 报告为独立文件（内联 CSS，可直接在浏览器打印/保存 PDF），包含元数据、风险 KPI 卡片、规则命中表 + 证据、网络证据

------

## 三、全局 UI 规范

| 元素 | 规范 |
|------|------|
| 主题色 | 薄荷绿（`--primary: #0d9488`），低饱和度背景 |
| 字体 | 系统字体栈：-apple-system, BlinkMacSystemFont, Segoe UI, Microsoft YaHei |
| 圆角 | 卡片 16px，按钮/输入框 12px，badge 999px |
| 导航栏 | sticky + 毛玻璃背景，品牌字号 24px/800，链接字号 15px/600 |
| Toast | 固定右上角，3 秒自动消失，slide-in/slide-out 动画 |
| 风险 badge | LOW=绿、MID=橙、HIGH=红，均为圆角 pill 样式 |
| 响应式 | ≤900px 导航堆叠、KPI 2列、Feature 2列；≤560px KPI/Feature 单列 |

所有 JS 内联在模板中（无独立 JS 文件），模板继承 `layout.html`（提供 nav、flash messages、footer）。
