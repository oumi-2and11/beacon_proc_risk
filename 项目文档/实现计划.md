# ProcRisk 项目实现计划

## Context

这是一个基于 Beacon 行为特征的主机进程风险检测系统课程作业项目。当前状态：
- ✅ 前端 UI 已优化（清新薄荷绿风格，7 个模板）
- ✅ 数据模型已定义完整（10 张 SQLAlchemy 表）
- ✅ Flask 应用框架已搭建（6 个蓝图）
- ✅ Docker + MySQL 开发环境已配置（端口 13306）
- ⏳ 所有核心逻辑均为 dummy 数据，需替换为真实实现
- ⏳ utils/ 目录存在但子模块为空（且部分目录命名有误）

设计决策：
- `utils/` 与 `webapp/` 同级，位于项目根目录
- 检测架构采用规则引擎模式（类注册 + 装饰器，非 YAML 配置）
- 中间数据使用 `ProcessInfo` dataclass 传递（非裸字典），避免模板数据格式不匹配

### models.py — ProcessInfo dataclass

```python
@dataclass
class ProcessInfo:
    pid: int
    name: str = ""
    exe_path: str = ""
    cmdline: str = ""
    username: str = ""
    ppid: int = 0
    parent_name: str = ""
    create_time: Optional[datetime] = None
    is_system: bool = False
    signed_status: str = "unknown"     # 匹配 DB enum
    process_key: str = ""
    connections: List[dict] = field(default_factory=list)
    cpu_percent: float = 0.0
    memory_rss: int = 0

    def to_dict(self) -> dict:
        """返回 processes.html 模板所需格式: {pid, name, user, path, ppid}"""
        return {"pid": self.pid, "name": self.name, "user": self.username,
                "path": self.exe_path, "ppid": self.ppid}
```

### windows.py 关键函数

| 函数 | 说明 |
|------|------|
| `collect_process_list(max_processes=5000)` | 用 `psutil.process_iter()` 枚举全部进程，返回 `List[ProcessInfo]` |
| `collect_single_process(proc, parent_cache)` | 从单个 psutil.Process 采集信息，AccessDenied/NoSuchProcess 返回 None |
| `collect_connections_for_pid(pid)` | 采集进程网络连接，返回 `[{protocol, local_ip, local_port, remote_ip, remote_port, state, seen_at}]` |

错误处理策略（对应 SPEC.md 3.1.3）：
- `AccessDenied`: 跳过该进程
- `NoSuchProcess`: 进程已退出，跳过
- `ZombieProcess`: 返回最小化 ProcessInfo

### rules.py — 规则体系

**核心类**:

```python
@dataclass
class RuleHitResult:
    rule_id: str
    title: str
    dimension: str       # "process" | "network" | "memory" | "other"
    score_delta: int
    evidence: Dict[str, Any]  # 存入 evidence_json

class BaseRule(ABC):
    rule_id: str = ""
    title: str = ""
    dimension: str = "other"
    default_weight: int = 10

    @abstractmethod
    def check(self, context: RuleContext) -> Optional[RuleHitResult]:
        """规则命中返回 RuleHitResult，未命中返回 None"""

# 注册表
_RULE_REGISTRY: List[type] = []

def register_rule(cls):
    _RULE_REGISTRY.append(cls)
    return cls
```

**4 个具体规则**:

| 规则 ID | 类名 | 维度 | weight | 检测逻辑 |
|---------|------|------|--------|----------|
| PROC-001 | `AnomalousParentChildRule` | process | 20 | 父进程名在 SUSPICIOUS_PARENT_CHILD 中且子进程在可疑列表 |
| PROC-002 | `SuspiciousPathRule` | process | 30 | exe_path 匹配 SUSPICIOUS_PATH_PATTERNS 且 signed_status != "signed" |
| PROC-003 | `UnsignedProcessRule` | process | 15 | signed_status == "unsigned" 且不在系统路径 |
| NET-101 | `BeaconingNetworkRule` | network | 35 | context.beacon_stats.suspected == True（由阶段 4 注入） |

### engine.py — RuleEngine

```python
class RuleEngine:
    def __init__(self):
        self.rules: List[BaseRule] = []
        self._init_default_rules()   # 实例化所有 @register_rule 注册的规则

    def load_db_config(self, db_session):
        """从 DB Rule 表加载配置：enabled=False 则移除，default_weight 不同则覆盖"""

    def evaluate(self, context: RuleContext) -> List[RuleHitResult]:
        """执行全部启用规则：先白名单过滤 → rule.check(ctx) → 收集命中"""

    def get_rule_by_id(self, rule_id: str) -> Optional[BaseRule]:
        """按 rule_id 查找规则实例"""
```

### beacon.py — Beaconing 检测

```python
@dataclass
class BeaconStats:
    sample_window: int = 0
    interval_mean_ms: Optional[float] = None
    interval_std_ms: Optional[float] = None
    interval_cv: Optional[float] = None
    suspected: bool = False
    jitter_like: bool = False
    notes: Optional[str] = None

def detect_beaconing(connections: List[dict], sample_window: int = 20) -> BeaconStats:
    """
    算法：
      1. 按 seen_at 排序连接
      2. 计算相邻连接间隔（ms）
      3. 不足 BEACON_MIN_SAMPLES → 不标记
      4. 计算 mean/std/cv = std/mean
      5. cv < BEACON_CV_THRESHOLD → suspected
      6. cv > BEACON_JITTER_CV_MIN 且 cv < 0.15 → jitter_like

    单次快照局限：psutil 是瞬时快照，所有 seen_at 相同。
    MVP 中以连接数启发式补充（同 IP 多连接 → 可疑），
    CV 算法完整实现，后续支持时序采样后即可生效。
    """
```

### scorer.py — 评分计算

```python
@dataclass
class ScoreResult:
    total_score: int = 0
    level: str = "LOW"
    score_process: int = 0
    score_network: int = 0
    score_memory: int = 0
    score_other: int = 0
    summary_reason: str = ""

def compute_score(hits: List[RuleHitResult]) -> ScoreResult:
    """
    算法：
      1. 按 dimension 分组求和 score_delta
      2. total_score = 所有 score_delta 之和，上限 100
      3. level = determine_risk_level(total_score)
      4. summary_reason = 主要命中规则标题，截断至 255 字符
    """
```

### detector.py — 主检测器

```python
@dataclass
class RemoteSummary:
    remote_ip: str
    remote_port: Optional[int]
    protocol: str = "TCP"
    conn_count: int = 0
    first_seen_at: Optional[datetime] = None
    last_seen_at: Optional[datetime] = None
    risk_hint: Optional[str] = None

@dataclass
class ConnectionDetail:
    protocol: str
    local_ip: Optional[str]
    local_port: Optional[int]
    remote_ip: Optional[str]
    remote_port: Optional[int]
    state: Optional[str]
    seen_at: datetime

@dataclass
class DetectionResult:
    process_info: ProcessInfo
    hits: List[RuleHitResult] = field(default_factory=list)
    score: ScoreResult = field(default_factory=ScoreResult)
    beacon_stats: Optional[BeaconStats] = None
    remote_summaries: List[RemoteSummary] = field(default_factory=list)
    connections: List[ConnectionDetail] = field(default_factory=list)

def detect_process(pid: int, db_session, sample_window: int = None) -> DetectionResult:
    """
    检测流水线：
      1. get_process_by_pid(pid) → 进程信息
      2. get_connections_for_pid(pid) → 网络连接
      3. 构建 RuleContext
      4. RuleEngine.evaluate(ctx) → 规则命中
      5. detect_beaconing(connections) → beacon_stats
      6. 将 beacon_stats 注入 context，触发 NET-101 规则
      7. compute_score(hits) → 评分
      8. 聚合远端 IP 汇总
      9. 返回 DetectionResult

    PID 不存在时抛出 ValueError。
    """
```
