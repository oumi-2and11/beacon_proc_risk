## 检测规则体系

### 5 条内置规则

| 规则 ID | 类名 | 维度 | 权重 | 检测逻辑 |
|---------|------|------|------|----------|
| PROC-001 | `AnomalousParentChildRule` | process | 20 | 父进程名在 `SUSPICIOUS_PARENT_CHILD` 中且子进程在可疑列表 |
| PROC-002 | `SuspiciousPathRule` | process | 30 | exe_path 匹配 9 种可疑路径模式 **且** signed_status != "signed" |
| PROC-003 | `UnsignedProcessRule` | process | 15 | signed_status != "signed" 且不属于系统路径 |
| PROC-004 | `SuspiciousNameRule` | process | 20 | 进程名匹配已知恶意模式（artifact/beacon/mimikatz 等 11 种）或随机 8+ 字母数字 .exe |
| NET-101 | `BeaconingNetworkRule` | network | 35 | beacon_stats.suspected == True（由 Beaconing 检测模块注入） |

### 可疑路径模式（9 种）

`\Temp\`、`\AppData\Local\Temp\`、`\Downloads\`、`\Public\`、`\Desktop\`、`\AppData\Roaming\`、`\ProgramData\`、`\PerfLogs\`、`\Recycle\`

### 可疑父子进程映射（6 组）

| 父进程 | 可疑子进程 |
|--------|-----------|
| svchost.exe | cmd, powershell, wscript, cscript |
| taskeng.exe | cmd, powershell |
| mshta.exe | cmd, powershell |
| winword.exe | cmd, powershell, wscript |
| excel.exe | cmd, powershell |
| outlook.exe | cmd, powershell |

## Beaconing 采样检测

### 多轮采样机制

**问题**：psutil 瞬时快照所有连接 `seen_at` 相同，CV 无法计算。

**方案**：`BeaconSampler` 以固定间隔（默认 5 秒）多次采集连接快照（默认 12 轮 = 60 秒），积累时间序列。

**核心概念**：

- **Check-in**：某轮新出现的远端连接 → Beacon 回连 C2
- **持久连接**：≥80% 采样轮次中持续存在的远端 → 长连接模式 Beacon

### CV 检测算法

1. 提取 `checkin_times`，计算相邻间隔
2. 间隔数 < 10 → 降级（弱信号：check-in 频率 ≥50% 也标记 suspected）
3. 计算 mean/std/CV = std/mean
4. CV < 0.3 → suspected；0.05 < CV < 0.15 → jitter_like

### 代理穿透

当远端 IP 为回环地址（127.0.0.1 等）时：

1. 遍历系统连接，找到 LISTEN 该端口的代理进程
2. 采集代理进程的远端非回环连接
3. 用真实远端替换回环地址，risk_hint 标注代理信息

### 回退启发式

采样不可用时：同 IP ≥ 2 连接 或 非标准端口 → suspected

### 参数配置（config.py）

| 参数 | 默认值 |
|------|--------|
| BEACON_SAMPLING_ENABLED | True |
| BEACON_SAMPLING_INTERVAL | 5 秒 |
| BEACON_SAMPLING_ROUNDS | 12 轮 |
| BEACON_MIN_SAMPLES | 10 |
| BEACON_CV_THRESHOLD | 0.3 |
| BEACON_JITTER_CV_MIN | 0.05 |

## 评分机制

**维度加法模型**：各规则 score_delta 按维度求和，总分上限 100。

**风险等级**：LOW(0-39) / MID(40-69) / HIGH(70-100)

**典型评分**：

| 场景 | 命中规则 | 总分 | 等级 |
|------|---------|------|------|
| 系统进程 svchost.exe | 白名单命中，跳过检测 | 0 | LOW |
| CS Beacon artifact_x64.exe | PROC-002(30) + PROC-003(15) + PROC-004(20) + NET-101(35) | 100 | HIGH |
| 用户目录普通程序 | PROC-003(15) | 15 | LOW |

## 异步扫描 API

单进程扫描采用异步模式，避免采样期间页面卡顿：

| 端点 | 方法 | 说明 |
|------|------|------|
| `/scans/api/start` | POST | 启动后台线程，返回 task_id |
| `/scans/api/progress/<task_id>` | GET | SSE 推送进度 |
| `/scans/api/status/<task_id>` | GET | 轮询状态（前端每秒调用） |

前端流程：点击扫描 → 调用 start → 展示进度条 → 轮询 status → 完成后跳转详情
